Async Event Driven Programming Model

Events of all types (mouse hover, click, input text, network response) are waiting around to triggered. Events can be tied to any html element. - Event Type: str that specifies event (most common events) - Event Target: the associated object. Like, a load event is associated to window as click is to a button tag - Event Handler/Listener: specify how to handle event. Need to register with a event type and event target through the web browser. - Event Object: Contains event specific details. - Event Propagation: events are delineated by where they are in the DOM tree. Event rules can be made by div membership for instance.

Event Categories

Limited Survey.. *Device-dependent input events Events tied to specific devices like mouse, keyboard, or touch (e.g., `mousedown`, `mousemove`, `keydown`, `touchstart`). Device-independent input events Events that represent actions regardless of input method, like `click`, `input`, and pointer events, working across mouse, keyboard, and touch. User interface events Higher-level events on form elements, such as `focus`, `change`, and `submit`, that represent interactions with UI components. State-change events Events triggered by browser or network changes, like `load`, `DOMContentLoaded`, `online`, `offline`, and `popstate`. API-specific events* Events defined by specific web APIs, such as media events (playing, volumechange), IndexedDB events (success, error), or XMLHttpRequest events.

Registering Event handler Property

*2 Ways of Registering* 1. (old) set a property of the document or object that is the event target. Limits one event target per each event type. 2. (new) pass the addEventListener method of the object or element. Any object can use the addEventListener() method with the arguments event type, callback function, and an optional third argument.
