variable - a user defined symbols that hold a reference to some value. Aside, Variable Declaration versus Initialization var - stores mutable references

Null Safety

Is a best practice that prevents the error(s) of 'unintentionally' accessing a variable set to null - this error is called a 'null dereferenced error' Dart compiler detects the possibility of the null dereferenced errors at compile time [Source - Dart Docs](https://dart.dev/language/variables#null-safety)

Nullability in 3 steps

  1. Must be use ?
String? name  // Nullable type. Can be null or string.

String name   // Non-nullable type. Cannot be null but can be string.
  1. Variable must be initialized before declaration
  2. You cannot access properties or call methods on an expression with nullable types
late

Use cases for late variables - Declaring a non-nullable variable that is initialized after its declaration

late String description;

void main() {
  description = 'Feijoada!';
  print(description);
}
  • Lazy initializing of a variable

In the following example if the variable temperature is never used then the function readThermometer is never called

// This is the program's only call to readThermometer().
late String temperature = readThermometer(); // Lazily initialized.

final and const

variables that never change. *Immutable* const 'compile-time constants' that will optimized runtime performance by storing the value in memory. Another way to say it, const variables must be determined at compile time

const variables and their values
var foo = const [];
final bar = const [];
const baz = []; // Equivalent to const []

regarding the foo variable, notice it has a const value, but foo itself is not a const variable. Therefore...

foo = [1, 2, 3];  // is valid
baz = [42]; // Error: Constant variables can't be assigned a value.
