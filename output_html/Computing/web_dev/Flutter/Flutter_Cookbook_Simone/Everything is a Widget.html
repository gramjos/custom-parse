Widget are classes that perform tasks on the user interface. Every visual item inherits from the widget class. Even the screen is a widget. - Text widgets display text - Padding widgets adds space between widgets - Scaffold widget provides structure for a screen All widgets together form the *Widget Tree (a nested tree data structure) Visual widgets come in two forms: stateless and stateful Stateless widgets are performant and immutable because there are no life cycles of states.The core of Stateless Widget the `build()` method. `build()` is called time a widget needs to be repainted (altered/changed) on the screen. When dealing with stateless widgets happens once at the start, if the device rotated and during a refresh(just like at the start) MaterialApp are the primary building block for following Google's _Material Design_ specification. Using the MaterialApp automatically implements other design standards. Regarding Material design jargon, the `MaterialApp`'s child is called a home and the `Scaffold`'s child is called a body.
